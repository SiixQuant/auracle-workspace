# SUMMARY
Nimbalyst is an Electron + React monorepo where packages/electron is the entire desktop app (main process, preload, renderer) and packages/runtime is a shared TypeScript/React *library* — NOT a local web server; the renderer is served by electron-vite's dev server on http://localhost:5273 in dev and loaded via loadFile() from out/renderer in production. The stack is React 18 + Lexical 0.44 (editor) + jotai (state) + Tailwind/CSS-variables (styling) + Vite 7/electron-vite (build), with vitest tests. Branding is deeply embedded: ~3,300 "Nimbalyst" occurrences across electron/runtime/cli/shared source, plus hardcoded phone-home endpoints — PostHog key phc_s3lQIILexwlGHvxrMBqti355xUgkRocjMXW4LjV0ATw (desktop, iOS, Android), electron-updater pinned to GitHub repo nimbalyst/nimbalyst, marketplace at extensions.nimbalyst.com, collab sync at wss://sync.nimbalyst.com, and Stytch B2B live public tokens. Sign-in exists (Stytch Google OAuth + magic links via the hosted "collabv3" Cloudflare Worker) but only gates optional collab/sync/sharing — there is no licensing or payment surface; signed-out is fully local. Packaging is electron-builder with Apple notarization (team 3GYP4YJ3DH) driven by .github/workflows/electron-build.yml. The `nim` CLI is an optional companion for the tracker system (loopback MCP-HTTP to a running app, or direct SQLite offline) and is not required.

# FINDINGS

## 1. App composition: electron vs runtime
packages/electron is the desktop app: main process at packages/electron/src/main (window mgmt in src/main/window/WindowManager.ts, IPC in src/main/ipc/, SQLite via better-sqlite3 + PGlite in src/main/database/, services in src/main/services/), preload at src/preload, and the React UI at src/renderer (plus tiny extra HTML entries src/session-manager/ and src/workspace-manager/). packages/runtime (@nimbalyst/runtime) is a shared LIBRARY, not a server: Lexical editor (src/editor), AI provider layer (src/ai/server/ProviderFactory.ts, providers/ClaudeProvider.ts, ClaudeCodeProvider.ts, OpenAICodexProvider.ts — this code executes inside the Electron MAIN process, reached from the renderer over IPC), sync clients (src/sync/), jotai stores, and UI components reused by the iOS/Android WebViews (packages/ios, packages/android). It is built as a library with `vite build` (packages/runtime/vite.config.ts) and aliased to source in dev by packages/shared/viteNimbalystPlugin.ts. packages/core has no package.json (placeholder); packages/shared is just that one vite plugin file.

## 1b. Ports/URLs and dev vs prod launch
NOT a local-web-server+browser architecture. Dev: `npm run dev` in packages/electron → scripts/dev.sh → build:worker + `electron-vite dev`; renderer served at http://localhost:5273 (port set in packages/electron/electron.vite.config.ts:346, VITE_PORT override; dev:user2 uses 5274 + NIMBALYST_USER_DATA_DIR). BrowserWindows loadURL http://localhost:${VITE_PORT||5273} in dev and loadFile(out/renderer/*.html) in production (src/main/window/WindowManager.ts:523-547, WorkspaceManagerWindow.ts:108-123, DatabaseBrowserWindow.ts:38-53). Real local servers that DO run: a loopback MCP HTTP server started at port 3456 (src/main/index.ts:2130 startMcpHttpServer(3456); binds 127.0.0.1 at src/main/mcp/httpServer.ts:1166; port walks upward if busy) whose endpoint descriptor is written to <userData>/mcp-endpoint.json (src/main/mcp/mcpEndpointDescriptor.ts) for coding agents and the nim CLI; plus src/main/mcp/extensionDevServer.ts and src/main/services/ai/claudeCliObservation/claudeApiProxy.ts. Production main entry: out/main/index.js (packages/electron/package.json "main").

## 2. Runtime tech stack
Framework: React 18 (peerDep in packages/runtime/package.json) with Meta's Lexical 0.44 as the rich-text engine and Monaco for code. State: jotai ^2.16 (runtime dep; JotaiProvider in packages/electron/src/renderer/index.tsx; docs/JOTAI.md; atoms in packages/electron/src/renderer/store/atoms/). Collab: yjs + y-websocket + y-monaco (packages/runtime/src/sync/CollabLexicalProvider.ts etc.). Styling: Tailwind CSS (root tailwind.config.ts, packages/electron/tailwind.config.ts, postcss.config.js) over a CSS-variable theme system (docs/CSS_VARIABLES.md, packages/electron/THEMING.md, single source packages/electron/src/renderer/index.css). Build: Vite 7 — runtime builds as a library (`vite build`, vite-plugin-dts), the app builds with electron-vite (packages/electron/electron.vite.config.ts). Tests: vitest 4 everywhere + Playwright e2e (packages/electron/playwright.config.ts). Markdown pipeline: react-markdown/remark-gfm, mermaid, shiki.

## 5. Account / sign-in / licensing surface
No licensing, payments, or gating of local features — LICENSING.md/README describe free+local (MIT). There IS an optional account system used only for collab/sync/sharing: src/main/services/StytchAuthService.ts implements Google OAuth + email magic link through the hosted 'collabv3' Cloudflare Worker (the sync server at sync.nimbalyst.com — its source is NOT in this repo; only wire types in packages/collab-protocol and docs/SYNC_JWT_MODEL.md); flow returns nimbalyst://auth/callback?session_token=…&session_jwt=…&org_id=… deep links, tokens stored via Electron safeStorage (stytch-accounts.enc). The JWT authenticates the y-websocket sync connection (SyncManager) and share-link creation (src/main/ipc/ShareHandlers.ts → https://sync.nimbalyst.com, links on share.nimbalyst.com). Sync is opt-in: src/main/services/SyncManager.ts:314 returns the plain local store unless getSessionSyncConfig().enabled; UI in src/renderer/components/GlobalSettings/panels/SyncPanel.tsx. Client-side encryption machinery exists for synced content (packages/runtime/src/sync/ECDHKeyManager.ts, TrackerEnvelopeCrypto.ts, src/main/services/OrgKeyService.ts, OrgAccessResolver.ts, TeamService.ts). Signed out, nothing contacts sync/auth servers; PostHog + update checks are the only default network traffic.

## 6. CLI package
packages/cli is `nim` (bin nim → dist/bin/cli.js), a companion terminal tool for Nimbalyst's tracker system (list/get/create/update/comment/archive tracker items, importers, sessions, docs) so terminal agents get the same tracker semantics as in-app agents. Two modes (packages/cli/README.md): LIVE — discovers a running app via <userData>/mcp-endpoint.json and talks to the loopback MCP-HTTP server; DIRECT — opens the better-sqlite3 DB read-only (guarded WAL writes only when no app is running, exit 5 if it would race). Deps: only better-sqlite3 + js-yaml. It is OPTIONAL — private (not published to npm yet; prebuilt-binary publishing listed as remaining Phase 4 work), nothing in the desktop app depends on it.

# RISKS
- Rebrand is large but mechanical in source (~3,327 'Nimbalyst' occurrences; 659 files in packages/electron/src + 217 in packages/runtime/src) — the hard parts are the functional identifiers: nimbalyst:// deep-link scheme (auth callbacks depend on it AND on the hosted collabv3 worker redirecting to it), com.nimbalyst.electron appId (changes userData path → data migration needed), .nimbalyst/ project folders, .nimext format, and the published @nimbalyst/extension-sdk npm scope.
- Auto-update feed is hardcoded to GitHub repo nimbalyst/nimbalyst (autoUpdater.ts:31-34); a fork that keeps this ships updates FROM UPSTREAM — it will silently replace a rebranded build with upstream Nimbalyst releases unless GITHUB_UPDATE_PROVIDER is changed first.
- Default phone-home even without sign-in: PostHog (us.i.posthog.com with the upstream project key baked into desktop, iOS, and Android) and hourly GitHub update checks; a rebrand that keeps the key sends your usage data to the upstream vendor's PostHog project. Opt-out exists but is on by default for official builds (OFFICIAL_BUILD=true).
- Extension marketplace and collab/sync depend on upstream-hosted infrastructure (extensions.nimbalyst.com Cloudflare Worker, sync.nimbalyst.com collabv3 worker, share.nimbalyst.com); the sync/auth server source is NOT in the repo — only the marketplace worker (packages/marketplace) is, so a self-hosted rebrand loses sign-in/sync/share unless that server is reimplemented (wire types are available in packages/collab-protocol, JWT model in docs/SYNC_JWT_MODEL.md).
- Stytch live public tokens are committed in packages/runtime/src/config/stytch.ts and point at the upstream vendor's Stytch project; any rebrand keeping them routes OAuth through upstream's tenant (env-var override STYTCH_PROJECT_ID/STYTCH_PUBLIC_TOKEN exists).
- Mac distribution requires the fork's own Apple Developer identity — team ID 3GYP4YJ3DH is hardcoded in build/notarize.js and .env.example and must be replaced along with the CSC_* / APPLE_* CI secrets in .github/workflows/electron-build.yml.
